# 手工题目计分改造计划

## Summary

- 目标：把当前“考试时按可答题数量均分 100 分”的计分方式，改造成“每个题目可手工指定分值；未指定时按题型默认分值补齐”。
- 计分规则来源：新增独立 JSON 配置文件，集中定义各题型默认分值与相关计分策略。
- 组合题规则：组合题父题不直接设分，子题单独设分，组合题总分等于各子题分值之和。
- 展示规则：考试模式同时保留“原始得分/卷面总分”与“折算后的百分制得分”；练习模式继续只显示对错统计，不新增总分展示。

## Current State Analysis

- 后端核心逻辑集中在 `d:\dev\family-exam-helper\app.py`。
- 当前 `questions` 表字段由 `_ensure_question_schema()` 建立，已包含 `type`、`difficulty`、`content`、`options`、`answer`、`explanation`、`parent_id`、`sort_order`、`stem`、`is_subquestion`，但没有题目分值字段。
- 当前录题与编辑逻辑：
  - 普通题通过 `_parse_question_form()` 解析，再由 `add_question()` / `edit_question()` 写入 `questions`。
  - 组合题通过 `_parse_composite_form()` 解析，父题与子题分别插入 `questions`。
- 当前导入导出逻辑：
  - JSON/ZIP 导出结构由 `_serialize_question_export()` 生成。
  - JSON/ZIP 导入由 `_import_question_payload()` 写入数据库。
  - Excel 示例文件头部仅包含 `科目/标签/题型/难度/题干/选项A-D/正确答案/解析`，没有分值列。
- 当前计分逻辑：
  - `_check_answer()` 只负责判定对错，不负责分值。
  - `_build_result_entry(question, user_answer, score_per_item=0)` 对普通题答对给 `score_per_item`，组合题对子题逐个按同一 `score_per_item` 计分并累加。
  - `exam_submit()` 先统计所有可答项数量，再计算 `score_per_q = 100.0 / total_items`，因此当前所有普通题和组合题子题权重完全相同。
  - `practice_result()` 只统计正确题/总题，不计算分数。
- 当前模板情况：
  - `d:\dev\family-exam-helper\templates\question_form.html` 提供题目录入/编辑界面，目前没有分值输入控件。
  - `d:\dev\family-exam-helper\templates\questions.html` 题目列表页目前展示题型、难度、组合题小题数量，不展示分值。
  - `d:\dev\family-exam-helper\templates\result.html` 考试模式当前直接显示 `score / total_score`，练习模式只展示正确题数；还没有“按卷面总分折算到 100 分”的单独显示口径。

## Proposed Changes

### 1. 新增独立计分配置文件

- 文件：`d:\dev\family-exam-helper\scoring_config.json`
- What：新增独立 JSON 配置文件，保存计分相关默认规则。
- Why：用户要求“计分规则单独设成配置文件”，并且当前项目还没有统一配置入口。
- How：
  - 定义配置结构，至少包含：
    - `default_scores_by_type`
    - `allow_manual_question_score`
    - `composite_scoring_mode`
  - 推荐落地结构：
    - `single_choice`
    - `multi_choice`
    - `true_false`
    - `fill_blank`
  - 初始默认值采用“按题型默认分值”，例如单选 2、多选 3、判断 1、填空 2；实际值由实现时写入该 JSON。
  - 配置加载逻辑放在 `app.py` 内集中封装，提供读取默认分值和兜底值的方法。

### 2. 扩展数据库结构以保存手工分值

- 文件：`d:\dev\family-exam-helper\app.py`
- What：为 `questions` 表增加题目分值字段，并兼容已有题库。
- Why：当前每题没有持久化分值，无法支持手工指定每题计分。
- How：
  - 在 `_ensure_question_schema()` 创建表时新增 `score REAL DEFAULT NULL` 或等效可空数字字段。
  - 在 `migrate_bank_schema()` 中加入旧库迁移逻辑，确保已有 `.db` 题库自动补列。
  - 约定字段含义：
    - `NULL` 表示未手工指定，运行时回退到配置文件默认值。
    - 非空数值表示显式手工分值。
  - 普通题与组合子题都使用同一个 `score` 字段。
  - 组合父题可保持 `NULL`，不作为独立计分项使用。

### 3. 封装统一的分值解析逻辑

- 文件：`d:\dev\family-exam-helper\app.py`
- What：新增“题目实际分值”解析函数，统一决定一题最终使用的分值。
- Why：后续新增录题、导入、考试汇总都需要同一套分值优先级，避免分散判断。
- How：
  - 新增类似 `_get_default_score_for_type(qtype)` 与 `_resolve_question_score(question)` 的辅助函数。
  - 分值优先级固定为：
    - 手工分值 `question.score`
    - 配置文件中的 `default_scores_by_type[qtype]`
    - 最后兜底常量
  - 对组合题：
    - 父题不单独取默认分值。
    - 子题逐个解析实际分值。
  - 必要时新增配置缓存或懒加载，避免每次请求重复读文件。

### 4. 改造录题与编辑表单

- 文件：`d:\dev\family-exam-helper\templates\question_form.html`
- 文件：`d:\dev\family-exam-helper\app.py`
- What：在题目录入与编辑界面增加“手工分值”输入，并支持组合题子题分值输入。
- Why：用户要求“手工指定每个题目的计分”，必须在管理端可编辑。
- How：
  - 普通题表单新增可选分值输入框，支持留空表示使用默认值。
  - 组合题父题不新增总分输入，避免与“子题单独设分”冲突。
  - 组合题小题编辑卡片新增分值输入项，并写入 `composite_payload`。
  - 在 `_parse_question_form()` 中解析普通题分值。
  - 在 `_parse_composite_form()` 中解析每个子题分值。
  - 在 `add_question()` / `edit_question()` 的 SQL 中写入 `score` 字段。
  - 对已有题目编辑时，需要把现有分值回填到模板。

### 5. 改造题目读取与结果组装

- 文件：`d:\dev\family-exam-helper\app.py`
- What：让题目对象和结果对象都带上“手工分值 / 实际生效分值”。
- Why：考试结果页、题目管理页以及后续排查都需要知道题目使用的是哪个分值。
- How：
  - 在 `_question_row_to_dict()` 输出中保留 `score` 原始字段。
  - 在 `_load_question_unit()` 加载组合题时，子题对象也保留各自分值。
  - 重写 `_build_result_entry()` 的计分方式：
    - 非组合题：答对得 `_resolve_question_score(question)`，答错得 0。
    - 组合题：对子题逐一按各自分值判分并累加。
  - 保留 `group_correct/group_total` 用于练习模式统计，不再依赖统一 `score_per_item`。

### 6. 重写考试总分汇总规则

- 文件：`d:\dev\family-exam-helper\app.py`
- What：移除 `exam_submit()` 中“均分到 100 分”的逻辑，改为按题目实际分值累加。
- Why：这是本次需求的核心改动点。
- How：
  - 去掉 `total_items -> score_per_q` 这条链路。
  - 新增两类汇总口径：
    - `correct_count`：仍按答对的可答项数量统计，用于保留原有“正确多少题”语义。
    - `total_score`：所有普通题和组合子题实际分值之和，作为卷面总分。
  - 考试得分 `score` 改为“答对题目实际得分之和”，作为原始得分。
  - 新增折算口径 `normalized_score = round(score / total_score * 100)`；例如原始得分 `180`、卷面总分 `223`，结果显示为“`180/223`，得分 `81` 分”。
  - `exam_records` 继续保存 `correct_count/score/total_score`，但其含义由“百分制”改为“题目分值累计制”。
  - 如需兼容历史记录显示，只在新记录中使用新口径，不回写旧记录。

### 7. 保持练习模式只看对错

- 文件：`d:\dev\family-exam-helper\app.py`
- 文件：`d:\dev\family-exam-helper\templates\result.html`
- What：练习模式沿用当前“正确题数/总题数”展示，不显示累计分数。
- Why：这是已确认的产品偏好，可减少界面扰动。
- How：
  - `practice_result()` 继续通过 `group_correct/group_total` 汇总正确数。
  - `result.html` 中练习分支保持不变，只在考试模式使用新的累计分值。
  - 如结果对象里新增 `effective_score` 等字段，练习分支不展示。

### 8. 补齐题目管理与可视化信息

- 文件：`d:\dev\family-exam-helper\templates\questions.html`
- 文件：`d:\dev\family-exam-helper\templates\result.html`
- What：在题目列表和考试结果中展示分值来源或分值结果。
- Why：手工分值上线后，管理员需要直观看到每题是多少分。
- How：
  - 在 `questions.html` 中为普通题显示“X 分”或“默认 X 分”。
  - 对组合题显示“共 Y 小题”外，可增加“合计 Z 分”或在展开子题时展示每小题分值。
  - `result.html` 考试模式继续显示每题/每小题分值，但数值来源改为题目实际分值。
  - 考试结果总览区改为同时展示：
    - 原始得分：`score / total_score`
    - 折算得分：`normalized_score` 分
  - 练习模式展示保持不变。

### 10. 增加容错与回退策略

- 文件：`d:\dev\family-exam-helper\app.py`
- What：为配置文件缺失、字段异常、非法分值等场景提供稳定回退。
- Why：计分配置外置后，运行时失败面会增加，必须有兜底策略。
- How：
  - 配置文件不存在或 JSON 解析失败时，回退到内置默认题型分值。
  - 手工分值若为空字符串或非法值，则按未指定处理。
  - 分值校验建议限制为正数，避免 0 分或负分题破坏结果统计。
  - 对组合题子题缺分值时，各自独立回退到对应题型默认分值。

## Assumptions & Decisions

- 决策：配置文件格式使用 `JSON`。
- 决策：默认分值按题型配置，而不是统一默认分或按难度默认。
- 决策：组合题采用“子题单独设分，父题不设总分”的模型。
- 决策：考试结果同时显示“原始得分/卷面总分”和“折算后的百分制得分”。
- 决策：折算后的百分制得分按四舍五入显示整数分。
- 决策：练习模式不显示累计得分，只保留正确题数展示。
- 假设：现有题库中的所有题目在新增 `score` 字段后，允许以 `NULL` 形式平滑过渡，不需要一次性批量回填历史分值。
- 假设：考试记录历史数据可以接受新旧语义并存；若后续需要统一展示口径，再单独做历史兼容改造。

## Verification Steps

- 检查 `questions` 表迁移后，新老题库都包含 `score` 字段，且旧题数据不丢失。
- 检查 `scoring_config.json` 在正常、缺失、格式错误三种情况下都能得到可用默认分值。
- 检查普通题：
  - 手工填分值时，考试结果按手工值计分。
  - 留空时，按题型默认值计分。
- 检查组合题：
  - 每个子题可独立设分。
  - 留空子题按其题型默认值计分。
  - 父题总分等于子题分值之和。
- 检查考试模式：
  - 原始得分 `score` 等于所有答对题目的实际得分之和。
  - `total_score` 等于试卷内所有题目的实际分值总和。
  - 折算得分 `normalized_score` 等于 `round(score / total_score * 100)`。
  - 当原始得分为 `180`、卷面总分为 `223` 时，结果页显示“`180/223`，得分 `81` 分”。
  - 结果页每题/每小题显示的分值与实际计算一致。
- 检查练习模式：
  - 仍只显示正确题数与总题数。
  - 不因新增分值字段改变原有答题流程。
- 检查导入导出：
  - JSON/ZIP 导出后包含分值字段。
  - 导入后分值保持一致。
  - Excel 示例模板包含“分值”列且说明清晰。
