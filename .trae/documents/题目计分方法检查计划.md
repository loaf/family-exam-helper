# 题目计分方法检查计划

## Summary

- 目标：按题型梳理当前项目中各类题目的实际计分方法，仅做现状检查与说明，不改动业务代码。
- 输出：一份面向当前代码实现的检查结论，覆盖单选题、多选题、判断题、填空题四类可答题型，以及组合题在总分汇总中的处理方式。
- 交付形式：基于代码事实给出“题型 -> 判题规则 -> 分值分配 -> 汇总方式 -> 展示方式”的结构化说明，并标注对应代码位置。

## Current State Analysis

- 项目核心逻辑集中在 `d:\dev\family-exam-helper\app.py`，题型定义、答案标准化、判题、结果组装、考试总分汇总均在此文件内完成。
- 题型常量位于 `d:\dev\family-exam-helper\app.py` 中的 `QUESTION_TYPE_LABELS` 与 `ANSWERABLE_TYPES`：
  - `single_choice`
  - `multi_choice`
  - `true_false`
  - `fill_blank`
  - `composite`
- 题目录入页面位于 `d:\dev\family-exam-helper\templates\question_form.html`，前端已暴露上述题型，且组合题说明中明确仅支持“单选、多选、判断、填空”作为小题。
- 判题入口为 `d:\dev\family-exam-helper\app.py` 中的 `_check_answer`：
  - 填空题：按 `|` 分隔多个可接受答案，使用不区分大小写的完全匹配。
  - 多选题：先标准化答案，再按集合全等判定，要求“全选对且不能多选”。
  - 单选题 / 判断题：按不区分大小写的完全匹配判定。
- 结果组装入口为 `d:\dev\family-exam-helper\app.py` 中的 `_build_result_entry`：
  - 非组合题：答对则得 `score_per_item`，答错得 0。
  - 组合题：对子题逐一判分并累加，组合题自身分数等于各子题得分之和。
- 练习结果汇总位于 `d:\dev\family-exam-helper\app.py` 中的 `practice_result`：
  - 只统计正确题/总题，不计算百分制得分。
- 考试结果汇总位于 `d:\dev\family-exam-helper\app.py` 中的 `exam_submit`：
  - 先按“所有可答题/子题数量”计算 `score_per_q = 100 / total_items`。
  - 每道普通题或每个组合子题都按同一权重计分。
  - 最终总分为 `correct * score_per_q`，并写入 `exam_records`。
- 结果展示位于 `d:\dev\family-exam-helper\templates\result.html`：
  - 考试模式显示总分与每题/每小题得分。
  - 练习模式显示正确题数，不显示百分制分值。

## Proposed Changes

### 1. 检查并说明题型定义来源

- 文件：`d:\dev\family-exam-helper\app.py`
- What：确认系统中实际参与计分的题型集合，以及组合题是否单独计分。
- Why：后续所有结论都必须先建立在真实题型定义上，避免把录入层支持的类型和计分层支持的类型混淆。
- How：基于 `QUESTION_TYPE_LABELS`、`ANSWERABLE_TYPES`、`_count_answerable_questions` 说明哪些题型直接计分、哪些题型只作为容器。

### 2. 检查并说明各题型的判题规则

- 文件：`d:\dev\family-exam-helper\app.py`
- What：逐项核对单选、多选、判断、填空的答案标准化和正确性判断规则。
- Why：用户要求“检查每个题目的计分方法”，判题规则决定“是否得分”，是计分逻辑的第一层。
- How：
  - 查看 `_parse_question_answer` 与 `_normalize_multi_answer` 的输入标准化方式。
  - 查看 `_check_answer` 中每种题型的比较方式。
  - 最终输出每个题型的“输入格式要求 + 正确判定条件”。

### 3. 检查并说明分值分配规则

- 文件：`d:\dev\family-exam-helper\app.py`
- What：核对普通题、组合题子题、组合题整体在考试模式下的分值计算方式。
- Why：这一步决定“答对之后拿多少分”，是用户最关心的计分核心。
- How：
  - 查看 `exam_submit` 中 `total_items` 与 `score_per_q` 的计算。
  - 查看 `_build_result_entry` 中普通题与组合题子题的 `score` 赋值方式。
  - 明确结论：当前系统并非按题型差异给分，而是按所有可答项均分 100 分。

### 4. 检查并说明组合题的特殊处理

- 文件：`d:\dev\family-exam-helper\app.py`
- 文件：`d:\dev\family-exam-helper\templates\question_form.html`
- What：确认组合题父题、子题在录入、判题、计分、汇总中的职责边界。
- Why：组合题是本项目中最容易让“每题怎么计分”产生歧义的部分。
- How：
  - 说明 `composite` 父题本身不直接作为一个固定分值题计分。
  - 说明父题总分来自子题逐个判分后的累计。
  - 说明父题“是否正确”取决于是否全部子题都答对，但该布尔值不改变已经累加的子题得分。

### 5. 检查并说明练习模式与考试模式的差异

- 文件：`d:\dev\family-exam-helper\app.py`
- 文件：`d:\dev\family-exam-helper\templates\result.html`
- What：确认两种模式下“计分”与“展示”的差别。
- Why：同一套判题逻辑在不同模式下的输出不同，若不拆开说明，容易误以为练习模式也存在百分制分数。
- How：
  - 对比 `practice_result` 与 `exam_submit` 的汇总字段。
  - 对比 `result.html` 中 `mode == 'exam'` 与其他模式的显示分支。
  - 形成结论：练习模式偏“对错统计”，考试模式偏“百分制记分”。

### 6. 产出最终检查结论

- 文件：`d:\dev\family-exam-helper\app.py`
- 文件：`d:\dev\family-exam-helper\templates\question_form.html`
- 文件：`d:\dev\family-exam-helper\templates\result.html`
- What：将以上检查结果整理成按题型分类的现状说明。
- Why：便于后续直接作为需求讨论、问题排查或改造的基线资料。
- How：输出时按以下结构组织：
  - 单选题
  - 多选题
  - 判断题
  - 填空题
  - 组合题（作为汇总规则补充说明）

## Assumptions & Decisions

- 决策：本次仅做“现状梳理”，不额外做问题审计或计分改造设计。
- 决策：输出粒度按“题型”组织，而不是逐条数据库题目实例展开。
- 假设：当前项目的真实业务实现以 `app.py` 为唯一后端入口，未存在其他隐藏计分服务或异步任务。
- 假设：模板文件仅负责展示，不包含会覆盖后端判分结果的前端二次计分逻辑。
- 决策：组合题虽然不在“按题型计分”的四类可答题型中单独参与均分，但必须作为总分汇总规则的一部分单独说明。

## Verification Steps

- 核对 `QUESTION_TYPE_LABELS`、`ANSWERABLE_TYPES`，确认题型范围。
- 核对 `_parse_question_answer`、`_normalize_multi_answer`、`_check_answer`，确认各题型判题规则。
- 核对 `_build_result_entry`，确认单题与组合题子题的得分生成方式。
- 核对 `exam_submit`，确认总题数口径、单题分值和总分计算公式。
- 核对 `practice_result` 与 `result.html`，确认练习/考试两种模式的展示差异。
- 最终输出一份按题型归纳的结论，所有结论都附着到已确认的代码路径，不引入代码外推测。
